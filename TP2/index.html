Compte rendu – TP2 : Présentation de l’environnement de travail VS Code

Lors de ce TP, j’ai découvert l’environnement de travail Visual Studio Code (VS Code). VS Code est un IDE (Environnement de Développement Intégré) qui permet de développer plus facilement grâce à des fonctionnalités comme la coloration syntaxique, l’auto-complétion, le débogage et la gestion des versions avec Git.

Dans un premier temps, j’ai créé un dossier nommé TP2-Web que j’ai ouvert avec VS Code. J’ai ensuite ajouté un fichier index.html afin de commencer un projet web en HTML.

Puis, j’ai installé l’extension Live Preview, certifiée par Microsoft. Cette extension permet d’afficher une page web en temps réel. J’ai appris à prévisualiser la page directement dans VS Code ou dans un navigateur, et j’ai pu voir les modifications s’actualiser automatiquement lorsque je changeais le code.

Ensuite, j’ai utilisé Git pour gérer les versions du projet. J’ai initialisé un dépôt local avec la commande git init. J’ai ajouté le fichier index.html au dépôt avec la commande git add, puis j’ai réalisé un premier commit avec le message First Commit.

J’ai aussi observé les différents états des fichiers : U pour non suivi, A pour ajouté et M pour modifié.

Par la suite, j’ai modifié le fichier index.html et ajouté un nouveau fichier test2.html. J’ai ensuite fait un nouveau commit et appris à revenir à une version précédente du projet grâce à la commande git reset --hard.

Enfin, j’ai créé un dépôt sur GitHub, que j’ai relié à mon dépôt local. J’ai envoyé mon projet en ligne avec la commande git push. Pour terminer, j’ai utilisé GitHub Pages afin de publier mon site web et obtenir un lien accessible en ligne.

Ce TP m’a permis de mieux comprendre l’utilisation de VS Code, de Git et de GitHub pour développer et publier un site web.
